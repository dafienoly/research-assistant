## 1. 总体结论

**当前系统处于「短样本探索级 · 价量因子为主 · 半导体主题研究初期」阶段，远未达到生产级候选或小资金实盘资格。**

**(a) 半导体研究主线的合理性**: ✅ 用户长期看好半导体/AI算力/国产替代方向，该判断主观层面合理，当前系统也建立了初步的半导体研究基础设施（315只标的面板、21只核心半导体标签、3只ETF数据）。但系统目前**无法证明半导体主题本身相对全A有超额**，更无法证明因子挖掘跑赢了半导体同池等权。

**(b) 300+股票池**: 当前 pool.csv 收录315只AI图谱标的，但数据源单一（仅来源于 aichainmap.com/atlas 的AI产业链图谱），口径极宽——包含了美的集团（L3 DCIM软件）、牧原股份（L5 AI垂直应用）、中国平安（L5其他AI应用）、莲花控股（算力租赁）等与半导体无实质关联的标的。这是一个**纯度低、未分层、不可交易映射的候选池**，不是可用的因子研究股票池。

**(c) 全A扩展必要性**: **必须扩展。** 当前没有全A基础行情数据，因子验证报告中"行业暴露"标注为"多行业分散（A股全市场）"但实际只有20只标的的截面数据。没有全A基础池就无法：
- 计算半导体相对全A等权超额
- 构建匹配对照池
- 做行业中性化
- 做市值/波动率暴露分析

**(d) 数据范围**: 个股日线行情仅覆盖 **2025-03-31 至 2026-04-03（约1年，247个交易日）**，仅6只股票。ETF数据同样仅1年。财务数据仅2025Q4和2026Q1两期。**这完全不足以做任何长期因子验证**——无法覆盖牛熊切换，无法捕捉半导体下行周期，2025年至今恰好是半导体/AI结构性牛市，所有因子都可能因为"在大涨行情中选到大涨股"而被误判为有效。

**(e) 因子体系**: 142个注册因子中，**约85%是价量技术指标变体**（动量、趋势、波动率、反转、成交量、技术形态）。基本面因子只有ROE、毛利率、净利率、负债率4个，且因子验证结果显示ROE和毛利率的IC为负（-0.016），Walk-forward验证未通过。LLM生成的候选因子（evolved_candidates.json 10个）全部是价量组合——缺少产业链标签因子、政策催化因子、事件驱动因子、海外映射因子。

**(f) 同池等权对比**: Factor Validation报告中已有"beats_peer"字段，但这里的"peer"是**全A股截面等权**，**不是半导体同池等权**。当前系统没有任何机制强制要求因子跑赢半导体同池等权——这意味着一个因子可能在全A中选到了半导体股（赶上板块Beta），但被标记为"beats_peer=True"，造成因子有效的假象。

**(g) 研究等级评定**: **短样本探索级（Short-sample Exploratory）**——不能标记为长周期验证级。

**(h) 小资金实盘**: **当前绝对不可进入实盘。** 阻塞项包括：无真实数据支撑、无法证明选股能力、无法剥离行业Beta、无交易约束验证、无账户权限映射。

**下一步最关键的3个任务**:
1. **建立U0全A基础池 + U3半导体核心池**——这是所有后续工作的数据底座
2. **扩展历史行情到至少2021年**——2025年以后数据不足以做任何长期判断
3. **建立半导体同池等权基准并强制作为因子晋级Gate**——解决"把Beta当Alpha"的根本问题

### 4.2 核心判断

**1. 因子严重偏价量**: 142个注册因子中，动量(6)+趋势(7)+波动率(6)+反转(5)+量价突破(12)+技术形态(12)+流动性(7)+ret5_penalty(5) ≈ **60个价量因子**，加上复合因子中的价量组合，约85%是价量指标变体。LLM evolution.py 生成的全部10个候选因子（evolved_candidates.json）也都是价量组合——说明LLM也只是一个价量公式生成器。

**2. 基本面因子验证失败**: 因子验证报告显示roe_q（IC=-0.0162）和gross_margin_q（IC=-0.0163）的IC为负且未通过Walk-forward。可能原因：(a) 仅1年数据不足以正确评估基本面因子；(b) 财务数据与行情日期不匹配（仅2期数据）；(c) 在牛市行情中，基本面因子收益率不如动量因子。

**3. 缺少产业假设到因子的闭环**: LLM生成的因子模板（evolution.py 第26-34行）只有6个价量模板。虽然有 llm_alpha_discovery.py 具备产业假设生成能力，但其 prompt 中 "Available Data Fields" 只有 close/open/high/low/volume/amount/returns/vwap —— 没有 pe_ttm, pb_lf, roe, revenue_growth 等字段，LLM不可能提出基本面或产业链因子。

### 核心判断

**"当前策略或因子是否已经强制和半导体同池等权比较？"**

**答案：否。** 当前 factor validation 中的 "beats_peer" 比较的是全A截面等权，不是半导体同池等权。这意味着：

- 一个超额收益为35.98%（如 vol_ratio5）的因子，可能只是因为选到了2025年大涨的半导体股，而不是因为量比因子本身在半导体内部能区分优劣
- 当前没有机制区分"因子选了正确的行业"和"因子在行业内部选了正确的股票"
- 因子晋级 Gate 仅要求 beats_peer=True + 综合评分达标，而 peer 不是半导体同池，这个 Gate 实际上是无效的

| AlphaLifecycle | ✅ | registry.py 有完整生命周期管理 | 正确但缺少与验证结果联动 | P2 |

### 核心判断

**当前 LLM 因子挖掘的本质是一个"价量公式生成器"**，不是真正的 Alpha 发现系统：
- LLM 的 prompt 中 Available Data Fields 只有 8个字段（close/open/high/low/volume/amount/returns/vwap），没有基本面/估值/资金流字段
- 生成的10个因子（evolved_candidates.json）全部是价量组合，无一涉及产业链、政策、事件
- 没有失败归因机制——当一个因子被淘汰后，LLM 无法知道"为什么失败"以及"应该朝什么方向改进"
- 没有实验次数记录——无法判断是第几次尝试、是否有数据挖掘偏差
- 因子模板只有6种价量模式

| P0.1 | **无全A基础行情数据** | daily_kline 目录仅6只股票，全A数据不存在 | 无法计算半导体相对全A超额、无法构建匹配对照池 | 永远无法证明因子不是行业Beta | 用 baostock/akshare 建立全A日线数据库 |

| P0.3 | **benchmark peer 定义错误** | "beats_peer" 比较的是全A截面，代码 validate_factor.py Line 194-249 | 让用户误以为因子已经跑赢半导体同池 | 因子晋级 Gate 形同虚设，Beta被当Alpha | 建立半导体300池等权和核心池等权基准 |
| P0.4 | **无半导体同池等权 Gate** | gate.py 没有半导体同池比较 | 因子可以从全A中选到半导体股但被标记为有效 | 策略和因子推荐系统出错的源头 | 因子晋级必须跑赢半导体核心池等权 |
| P0.5 | **因子体系85%偏价量** | factor_base.py 中价量因子占绝对多数 | 无法捕获半导体产业链的核心驱动因素（政策、订单、扩产、国产替代） | 因子库增量失效，策略同质化 | 补充半导体专属因子（见V3.3） |
| P0.6 | **缺少未来函数检查** | validate_factor.py 和 llm_alpha_discovery.py 均无未来函数检测 | LLM生成的因子可能隐含未来信息（如使用全日行情判断收盘信号） | 因子IC可能是虚假的 | 加入未来函数检测gate |

## 10. 最终判断

### Q1: 当前是否应该继续以半导体作为核心研究主线？
**是。** 半导体/AI算力/国产替代是中长期产业趋势，作为研究主线主观合理。但当前系统无法客观证明这条主线有效，需要先建立数据基础设施。

### Q2: 当前300多只股票池是否应该保留？
**保留作为 U2（广义AI候选池），但必须分层。** 当前池口径太宽（含牧原股份、中国平安等），必须拆出 U3 核心半导体池。同时需要标记来源置信度，补充多信源交叉验证。

### Q3: 是否必须扩展全A数据？
**是——这是 P0。** 没有全A基础数据，连"半导体相对全A是否有超额"这个最基本的问题都无法回答。建议使用 baostock（可免费获取2018年至今全A行情）。

### Q4: 当前2025年以后数据能支持什么结论，不能支持什么结论？
**能支持**: 2025年以来半导体/AI主线的短期趋势和价量关系描述（约1年行情）。
**不能支持**: (1) 不能证明因子长期有效；(2) 不能证明因子在熊市/震荡市有用；(3) 不能证明因子选择能力大于行业Beta；(4) 任何关于"年化XX%超额"的结论都是不可靠的——因为样本内只有1年。
**当前只能标记为"短样本探索级(Short-sample Exploratory)", 不能标记为"长周期验证级"。**

### Q5: 当前因子是否证明了选股能力，还是只吃到了行业Beta？
**当前无法区分。** 因子验证报告中的"beats_peer"比较的是全A截面，而不是半导体同池。如果一个因子的超额收益来自"在全A中选到了2025年大涨的半导体股"，这个因子的选股能力在半导体内部可能是负的。**当前系统完全没有能力区分"选了好的行业"和"在行业内部选了好的股票"**。

### Q6: 当前是否已经跑赢半导体同池等权？
**无法回答。** 当前系统连"半导体同池等权"这个基准都不存在。即使有（需要V3.4完成），当前只有6只股票的日线数据，根本无法构建有意义的半导体同池等权。

### Q7: 当前是否可以进入小资金实盘？
**绝对不可。** 见第8节 P0 问题——无数据支撑、无能力证明选股、无行业Beta剥离、无交易成本估计、500+阻塞项。

### Q8: 如果不能，最少需要补齐哪3个P0能力？
1. **U0全A基础池 + U3半导体核心池**（V3.1）
2. **历史数据扩展到至少2021年**（V3.2）
3. **半导体同池等权基准 + 强制Gate**（V3.4）

### Q9: 下一版最应该优先开发什么？
按顺序：**V3.1股票池分层 → V3.2数据扩展 → V3.4基准体系 → V3.3半导体因子 → V3.5评价增强 → V3.6 LLM升级 → V3.7实盘前置**

### Q10: 哪些当前模块可以保留，哪些必须重构？

**可保留（结构合理）**:
- Factor Engine 架构（factor_base.py + factor_engine.py）
- Alpha Registry（alpha/registry.py + schema.py 生命周期管理）
- AShareBacktester（execution_aware_backtester.py 交易约束正确）
- PortfolioBacktestEngine（portfolio_backtest.py 组合回测正确）
- GateEngine（core/gate.py 门禁设计合理）
- KillSwitch + Risk Sentinel（风控框架设计良好）
- Factor Validation 流程（validate_factor.py 评价框架合理，但需修复 peer 定义）
- Benchmark 模块（portfolio/benchmark.py 指数获取正确，缺自定义基准）
- Paper Trading（standing_paper_trading.py）

**必须重构**:
- **数据层(P0)**: 需要重构整个数据管线，从"手动CSV文件"升级为"可编程全量拉取+增量更新"
- **股票池定义**: 从单层 pool.csv 重构为5层分层体系（U0-U4）
- **因子类别扩展**: 从14个类别扩展包括主题择时/半导体专属/风险暴露
- **系数验证 peer 定义**: "beats_peer" 必须改为 beats_semiconductor_peer
- **LLM prompt 升级**: Available Data Fields 必须扩展
- **LLM evolution 流程**: 加入失败归因和迭代闭环
