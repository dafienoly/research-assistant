## Implementation

The V7.5 Report Center provides a unified API for discovering, viewing, and managing all reports generated by the Hermes research system. It follows the 报告中心 (Report Center) design with 6 REST API endpoints covering 5 report types.

### API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/reports/health` | 健康检查 — reports base directory status |
| `GET` | `/api/reports/summary` | 概览统计 — total count by type, recent 7d, total size |
| `GET` | `/api/reports` | 报告列表 — type filter, pagination (limit/offset), sorting |
| `GET` | `/api/reports/detail/{type}/{id}` | 详情查看 — full content of any report |
| `DELETE` | `/api/reports/{type}/{id}` | 删除报告 — type-aware delete (dir/file) |
| `GET` | `/api/reports/recent` | 最近报告 — time-window filter (hours) |

### Report Types

| Type | Source Directory | Description |
|---|---|---|
| `backtest` | `backtests/<factor>/` | metrics.json + report.html + returns.csv |
| `strategy` | `strategies/<group>/` | HTML strategy reports |
| `version` | `version_reports/` | Version completion JSON reports |
| `session` | `session_backups/<sid>/` | Agent session backups (request/summary/answer) |
| `roadmap` | `roadmap_backups/<backup>/` | Roadmap backup snapshots |

### Key Features

- **Auto-discovery**: Scans the HermesReports directory structure on each request — no registration needed
- **Intelligent parsing**:
  - Version reports auto-detect `completion_` vs `version_report_` prefix
  - Skips `latest.json` to avoid duplicates
  - Backtest reports parse `metrics.json` for sharpe/cagr/max_drawdown
  - Strategy reports extract metadata from filenames
- **Safe deletion**: Directory-level delete uses `shutil.rmtree`, file-level uses `unlink`, with error handling
- **Flexible time parsing**: Tolerates ISO 8601, custom formats, and regex-based timestamps
- **Resilient**: Degrades gracefully when the reports base directory doesn't exist
- **Configurable**: Reports base path via `HERMES_REPORTS_BASE` env var (default: `/mnt/d/HermesReports`)
