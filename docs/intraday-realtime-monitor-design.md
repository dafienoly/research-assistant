# 盘中实时监测设计文档

## 概述

本文档定义 Hermes 盘中高频实时监测的架构设计、监测规则、预警流程。

## 架构

```
┌──────────────────────────────────────────────────────────┐
│                   定时触发器 (cron)                         │
│  30-60s 调用 intraday:check-once                         │
└────────────────────┬─────────────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────────────┐
│              intraday-realtime-monitor                     │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────────┐  │
│  │ 数据读取模块   │  │ 规则引擎     │  │ 事件分类器     │  │
│  │ - live_snap   │  │ - 涨跌幅规则  │  │ L0-L4 分级     │  │
│  │ - 分钟线      │  │ - 成交额异常  │  │ - 推送分流     │  │
│  │ - 板块指数    │  │ - 板块共振   │  │ - 升级决策     │  │
│  └──────────────┘  └──────────────┘  └────────────────┘  │
└────────────────────┬─────────────────────────────────────┘
                     │
        ┌────────────┼────────────┬────────────┐
        ▼            ▼            ▼            ▼
┌────────────┐ ┌──────────┐ ┌──────────┐ ┌────────────┐
│ events_log │ │ dedup    │ │ wechat   │ │ escalation │
│ (所有级别)  │ │ (去重冷却) │ │ (推送)    │ │ (Codex 门) │
└────────────┘ └──────────┘ └──────────┘ └────────────┘
```

## 监测优先级 P0-P5

| 等级 | 对象 | 来源 | 频率 | 告警阈值（默认） |
|------|------|------|------|-----------------|
| P0 | 持仓股 | positions.csv | 30-60s | 跌幅>3%→L2, >5%→L3 |
| P1 | 今日推荐池 | today_candidates.csv | 30-60s | 涨幅>7%→L2追高风险 |
| P2 | 手动关注池 | watchlist.csv | 30-60s | 跌幅>4%→L2, 多规则共振→L3 |
| P3 | 重点产业链 | tags/*.csv | 1-3min | 板块同步>3%→L2 |
| P4 | 板块指数 | 申万一级/ETF | 30-60s | 板块跌幅>2%→L1 |
| P5 | 全 A 快照 | live_snapshot | 5-10min | 数据延迟>60s→L0 |

## 规则引擎

### 默认规则

| ID | 规则 | 条件 | 级别 | 备注 |
|----|------|------|------|------|
| R01 | 持仓股大跌 | 跌幅 > 3% | L2 | 仅 P0 |
| R02 | 持仓股暴跌 | 跌幅 > 5% | L3 | 仅 P0 |
| R03 | 推荐池追高 | 涨幅 > 7% | L2 | 仅 P1 |
| R04 | 关注池大跌 | 跌幅 > 4% | L2 | 仅 P2 |
| R05 | 板块同步跳水 | 板块内 > 3 只跌幅 > 3% | L2 | P3-P4 |
| R06 | 板块暴跌 | 板块内 > 5 只跌幅 > 5% | L3 | P3-P4 |
| R07 | 成交额异常 | 当前成交额 > 5日均值 500% | L1 | 全池 |
| R08 | 数据延迟 | live_snapshot 延迟 > 60s | L0 | 标注禁止强判断 |
| R09 | 数据缺失率高 | live_snapshot 某字段缺失 > 20% | L1 | |
| R10 | 尾盘限制 | 14:55-15:00 | - | 禁止新开仓类 alert |

### 规则可配置

规则阈值通过 `intraday/alert_state.json` 中的 `rules` 段可调：

```json
{
  "rules": {
    "R01": {"threshold": 3.0, "level": "L2"},
    "R02": {"threshold": 5.0, "level": "L3"},
    "R03": {"threshold": 7.0, "level": "L2"}
  }
}
```

## 事件分级 L0-L4

| 级别 | 名称 | 推送 | Codex 升级 | 说明 |
|------|------|------|------------|------|
| L0 | silent_log | ❌ | ❌ | 只写 events_log.jsonl |
| L1 | watch | digest（摘要） | ❌ | 进入观察池 |
| L2 | user_notice | 即时推送 | ❌ | 直接推企业微信 |
| L3 | codex_review | 即时推送 | ✅ | 推微信 + 上报 Codex |
| L4 | critical | 立即强提醒 | ✅ | 不受预算限制 |
