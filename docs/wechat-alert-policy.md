# 企业微信预警策略

## 概述

本文档定义盘中预警向企业微信推送的策略、格式规范和约束。

## 推送分流

| 事件等级 | 推送通道 | 说明 | 格式 |
|----------|----------|------|------|
| L0 | wechat_silent | 不推送，只记录 | - |
| L1 | wechat_digest | 进入摘要，定时推送 | 摘要卡片 |
| L2 | wechat_notice | 即时轻量推送 | 文本消息 |
| L3 | wechat_notice | 即时推送 + 上报 Codex | 文本消息 |
| L4 | wechat_urgent | 立即强提醒 + 上报 Codex | Markdown 卡片 |

## 配置与环境变量

```bash
# 是否启用企业微信推送
WECHAT_ENABLED=false
# 企业微信机器人 webhook URL
WECHAT_WEBHOOK_URL=
# dry-run 模式（只记录不推送）
WECHAT_DRY_RUN=true
```

dry-run 模式下：
- 写入 `wechat_push_log.jsonl`（sent: false）
- 不真实调用 webhook

## 推送频率控制

| 通道 | 频率限制 | 超限处理 |
|------|----------|----------|
| wechat_notice | 同一股票 5min 内不重复 | 进入 digest 等待下个窗口 |
| wechat_urgent | 同一股票 1min 内不重复 | 忽略（除严重程度升级） |
| wechat_digest | 每 10min 或每 20 条发送一次 | 自动合并 |

## 禁止用语

推送内容中不得出现：

- ❌ 立即买入 / 立即卖出
- ❌ 满仓 / 梭哈 / 清仓
- ❌ 必须止损 / 必须止盈
- ❌ 确定反转 / 确定出货 / 确定洗盘
- ❌ 主力 / 庄家 / 拉升 / 打压（非事实描述）

## 允许用语

- ✅ 建议观察 / 暂缓追高
- ✅ 进入风险观察 / 等待确认
- ✅ 需要 Codex 复核 / 需要人工确认
- ✅ 数据异常，禁止强判断
- ✅ 建议评估 / 建议关注
