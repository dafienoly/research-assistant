# ADR-023: 自动因子挖掘管线

**状态**: 已采纳  
**日期**: 2026-07-07  
**上下文**: 因子挖掘工作流（research:loop/factor:evolve → factor:mine → backtest:factor-top → factor:batch → alpha:register）目前缺自动衔接环节，IC评估与组合回测之间需手动衔接。  
**动机**: 实现无人值守的日频全自动因子挖掘——收盘后自动从新因子发现到注册到 Alpha Factory。

---

## 决策

### 1. 自动化范围 — 全自动闭环

从因子表达式产生到 Alpha 注册全自动，人类只在审批阶段介入。

```
因子设计 → IC评估 → 组合回测 → 稳健性检查 → 影子观察 → Alpha注册 → 通知
  (auto)     (auto)      (auto)      (auto)      (auto)    (auto)   (push)
                                                                       │
                                                                ┌──────┴──────┐
                                                                │  企微通知    │
                                                                │ (完整表格)   │
                                                                └─────────────┘
```

### 2. 触发方式 — 定时 + 事件双触发

- **定时**: 每日 16:00 cron（收盘后），处理当天 `evolved_candidates.json` 中的新因子
- **事件**: `research:loop` 或 `factor:evolve` 产出新表达式后，自动将表达式写入队列，由 worker 消费

### 3. 回测分级策略

| IC 范围 | 处理方式 |
|---------|----------|
| \|IC\| ≥ 0.03 | 完整验证（walk-forward + 参数敏感性 + 抗过拟合） |
| 0.015 ≤ \|IC\| < 0.03 | 快速回测（Top20% 分位，月频） |
| \|IC\| < 0.015 | 跳过，记录日志 |

### 4. 因子候选来源

全量注册表（~142 因子）+ `evolved_candidates.json`。已在注册表中的因子不重复验证，仅标记"上次验证时间"。

### 5. IC 阈值 — 固定绝对值两档

`|IC| = 0.03` / `0.015` 硬阈值，不随因子池分布变化。

### 6. 注册条件 — 自动注册

完整验证通过（Sharpe > 1.0, MaxDD < -20%, Walk-Forward pass）→ 自动注册到 Alpha Registry。

### 7. 通知方式 — 全量推送企微

IC 结果 + 每因子的回测摘要（含 Sharpe/MaxDD/IC/ICIR）→ 企微群。失败因子附原因。注册待确认消息独立推送。

### 8. 回测股票池 — 全量 watchlist

~300 只 watchlist 股票，跟 `factor:mine` 使用的同一池子。

### 9. 失败重试 — 重试 1 次 + 企微告警

每环节失败后自动重试 1 次。二次失败 → 推送到企微告警（含失败原因 + 异常堆栈），管线继续跑其它因子。

### 10. 持仓周期 — 跟因子走

因子定义中配置 `holding_period` 参数，回测时按因子自身的持仓周期执行。默认 5 日。

### 11. 时间窗口 — 分段验证

前 12 个月（2025-01 ~ 2025-12）用于 IC 计算和训练，后 6 个月（2026-01 ~ 2026-06）用于样本外回测。

### 12. 表达式设计 — research:loop 自主循环

6 阶段循环（生成 → 变异 → 交叉 → IC 评估 → 记录 → 迭代），`factor:evolve` 作为快速批量出候选的补充渠道。

### 13. 管线时序 — 异步队列

```
16:00 cron
   ↓
factor:mine（IC 评估，串行）
   ↓
结果入队（按 IC 分级标注处理类型）
   ↓
worker 1: backtest:factor-top（消费快速回测任务）
worker 2: 完整验证（消费完整验证任务）
worker 3: alpha:register（消费验证通过任务）
   ↓
所有 worker 完成后 → 汇总报告 → 企微推送
```

`factor:mine` 不阻塞，快速完成后即返回。回测和验证在后台 worker 中异步执行。

### 14. 影子观察期 — 按因子特性定

- 日频因子（默认）：10 个交易日
- 周频因子（`holding_period ≥ 5`）：1 个月
- 观察期内记录逐日 IC，期终计算 IC 衰减率
- 衰减率 < 30% → 标记为可用；≥ 30% → 标记为不稳定，通知人工审查

### 15. 推送细度 — 完整表格

企微推送包含以下信息的表格：
- 因子名 | IC | ICIR | Sharpe | MaxDD | 分级 | 状态
- 失败因子单独列出，附失败原因

---

## 影响

**正面**:
- 收盘后无人值守自动运行因子挖掘全流程
- IC 评估 → 回测的衔接不再需要手动操作
- 新因子从发现到注册的延迟从数小时降至分钟级

**负面**:
- 异步队列增加了系统复杂度，需要 worker 状态管理
- 影子观察期推迟了因子可用时间（日频 10 天）
- 全量推送企微可能产生较多消息（首次运行时 142 因子全量评估）

## 实现优先级

P0: 异步队列 + worker 框架 + IC→回测衔接
P1: 影子观察器 + 自动注册
P2: 企微推送全量表格
