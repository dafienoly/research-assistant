# ADR-022: ETF 跳水预测系统设计

**状态:** 已采纳
**日期:** 2026-07-07
**领域:** 盘中实时监测 / 预测

## 背景

原有 `etf-dive-warning` 仅做午盘单次规则评分，缺乏全天预测能力和历史验证。用户需要真正的**预测系统** — 在跳水发生前发出预警，而非事后通知。

## 需求

| 维度 | 决定 |
|------|------|
| 下跌定义 | 跌幅≥4% / 跌破分时均线 / 5分钟跌速>3%（任一触发） |
| 时间窗口 | 全天分段：开盘→午盘修正→实时15min滚动 |
| 行动阈值 | 下跌概率>60% 建议减仓/止损 |
| 监测范围 | 159516 + 板块指数 + 龙头个股先行信号 |
| 预测频率 | 15min滚动 + 事件驱动（龙头异动/大盘急跌） |
| 预测模型 | 规则引擎 → 历史回测验证 → XGBoost/LSTM |
| 计算资源 | 本地 + LLM辅助新闻情绪分析 |
| 推送格式 | 详细报告（信号明细 + 概率 + 数据） |

## 架构

```
数据层 (akshare + snapshot + mx:search)
    ↓
特征工程 (量价/资金/板块/情绪)
    ↓
信号引擎 (规则 + ML 融合)
    ↓
概率输出 + 主动推送 (企业微信)
```

## 关键设计决策

1. **数据源优先级:** akshare(fund_etf_spot_em) → 本地快照 → mx:data
2. **无代理依赖:** akshare 部分接口走东方财富 API，需 unset proxy
3. **回测周期:** 250个交易日，验证每个信号的历史预测准确率
4. **ML推进路径:** 先规则+回测跑通 → 积累足够特征后上XGBoost → LSTM做时序
