V3.2 **Factor Evaluation & Orthogonality** completed on 2026-07-05.

**Deliverables:**
- `factor_lab/factor_evaluation.py` — New unified API module with `FactorEvaluation` class and `run_full_evaluation()` shortcut, wrapping:
  - `evaluate_ic()` — IC/ICIR/daily IC/layer test via `ic_analyzer.py`
  - `evaluate_anti_overfit()` — Anti-overfit diagnostics via `validation/anti_overfit.py`
  - `evaluate_walk_forward()` — OOS Walk-Forward via `walk_forward.py` / `validation/rolling_validator.py`
  - `evaluate_orthogonality()` — Orthogonality + incremental value via `orthogonality/orthogonality_analyzer.py`
  - `evaluate_scoring()` — Factor scoring via `scoring/factor_score.py`
- `tests/test_factor_evaluation.py` — 13 integration tests for the evaluation API

**Pre-existing V3.2 components** (all 394 tests passed before changes):
- `ic_analyzer.py` — IC, daily IC, RankIC IR, layer test
- `validation/anti_overfit.py` — IC stability, stress test, placebo, IC decay, peer benchmark
- `walk_forward.py` — Full walk-forward with PSR/DSR
- `validation/rolling_validator.py` — Rolling train/val/test validation
- `orthogonality/orthogonality_analyzer.py` — Pearson/Spearman/TopN overlap/incremental value
- `orthogonality/ret5_filter_validator.py` — 6 filter strategies
- `scoring/factor_score.py` — Comprehensive factor scoring with family awareness
- `validate_factor.py` / `validate_orthogonality.py` — CLI entry points

**Test results:** 408 passed (394 original + 13 new), 0 failed.

**Why:** V3.2 brings IC/ICIR/OOS/Walk Forward and Orthogonality into a single, consistent, importable API. Previous code was scattered across modules with no unified entry point.

**How to apply:** Use `from factor_lab.factor_evaluation import run_full_evaluation, FactorEvaluation`. The class supports both `run_full_evaluation()` one-shot and individual step methods (`evaluate_ic()`, `evaluate_anti_overfit()`, etc.) for fine-grained control.
